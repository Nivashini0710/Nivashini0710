I'm an ECE undergraduate who got tired of choosing between "hardware person" and "software person," so I stopped choosing. My work lives at the point where a radiation pattern in MATLAB becomes a real antenna array, where an RTL block becomes a working silicon-level datapath, and where a neural network becomes a decision a drone makes mid-flight.

I completed an **internship at DRDO's Defence Electronics Research Laboratory (DLRL)**, in the **Antenna Wing**, working on **phased array antenna simulation** and **VNA-based RF measurement** — real antenna theory validated against real hardware, not just textbook equations. On the digital design side, I've built and optimized **cryptographic hardware in Verilog**, published research from that work, and compete in hackathons where the constraint is always the same: build something that actually works, in a weekend, under pressure.

I think in **systems**: sensors → signal → intelligence → action, and increasingly, **architecture → RTL → silicon**. ### 📡 Phased Array Antenna Simulation & Beam-Steering Suite
**Modeling how a 16-element array "aims" energy without moving a single part.**

`MATLAB` `Array Signal Processing` `RF Engineering` `Numerical Methods`

- Simulated a **16-element Uniform Linear Array (ULA)** at **f = 10 GHz**, `d = 0.0098 m` spacing, computing directivity and gain via numerical integration (`trapz`).
- Implemented **electronic beam steering** across broadside and endfire configurations by controlling progressive inter-element phase shift (β).
- Implemented the **Hansen–Woodyard endfire condition** for enhanced directivity, and diagnosed multi-lobe / grating-lobe behavior caused by spacing constraints near true endfire.
- Built **2D polar and 3D radiation pattern** visualizations, and combined **pattern multiplication** with half-wave dipole element patterns — including handling singularities that produced `NaN` values in the dipole equations.
- Directly extends theory from antenna array design work carried out during an internship at **DRDO's Defence Electronics Research Laboratory, Antenna Wing**.

**Why it matters:** This is the same class of math that drives radar, 5G massive MIMO, and satellite communication front-ends — implemented and debugged from first principles, not from a library call.

### 🔐 Hybrid-Adder-Optimized RSA Cryptographic Core (Verilog)
**A hardware-level RSA implementation, redesigned for speed and area — with the results written up as a conference paper.**

`Verilog HDL` `VLSI` `Cryptographic Hardware` `Xilinx Vivado` `Cadence Genus`

- Implemented the **RSA public-key algorithm** at the RTL level in Verilog, including the modular exponentiation datapath that dominates RSA's compute cost.
- Designed and integrated a **hybrid adder architecture** (combining fast and area-efficient adder structures) to optimize the critical path in the modular multiplier, improving speed/area/power trade-offs over a standard adder-based baseline.
- Verified functional correctness and synthesized the design, evaluating results against conventional adder implementations.
- **Authored and submitted a paper on this work to an academic conference** — turning an RTL optimization project into peer-reviewed research.

**Why it matters:** RSA hardware acceleration sits at the core of secure communication chips — showing you can take a cryptographic algorithm from math to an optimized, synthesizable circuit is a strong VLSI + security signal.
